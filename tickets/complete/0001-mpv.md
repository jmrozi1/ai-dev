TICKET: Add Terminal-Based AI Dev Chat Shell MVP

SUMMARY

Add a new terminal-based AI Dev assistant experience in VS Code.

The MVP should establish the interaction model only:

- Persistent terminal session
- Chat input at the bottom
- Slash-command mode
- Command discovery and completion
- Ephemeral progress/status output
- Permanent final transcript output
- Model startup feedback
- A top-level AI Dev launcher in the VS Code left Activity Bar

This ticket does not need to add advanced project routing, Jenkins intelligence, report handling, or a broad command set. The goal is to create the terminal shell that future AI Dev capabilities will use.

USER STORY

As a developer, I want to launch AI Dev from the VS Code left Activity Bar and interact with it as an ongoing terminal chat session, so that asking questions and running AI Dev commands feels natural, persistent, and observable without opening separate input boxes or large reports.

ACCEPTANCE CRITERIA

Activity Bar launcher

1. A top-level AI Dev item is visible in the VS Code left Activity Bar.
2. The launcher is presented alongside other top-level VS Code entries such as Explorer, Search, Source Control, Run and Debug, Extensions, Accounts, and Settings.
3. Selecting the AI Dev Activity Bar item opens or reveals the AI Dev experience.
4. The Activity Bar item uses an AI Dev icon consistent with the extension.
5. The launcher remains available after VS Code restarts while the extension is installed and enabled.
6. The launcher does not require a workspace-specific .ai-dev.yaml file to appear.
7. If no workspace is open, selecting the launcher shows a clear message or disabled state rather than failing silently.
8. The Activity Bar launcher is the primary discoverable entry point for the terminal assistant.
9. The command palette entry "AI Dev: Launch Assistant" remains available as an alternate launch path.
10. If an AI Dev terminal session already exists, selecting the Activity Bar launcher reveals and reuses it instead of creating a duplicate.

Launch and startup

11. A VS Code command named "AI Dev: Launch Assistant" is available.
12. Running the command opens or reveals a terminal named "AI Dev".
13. If the AI Dev terminal already exists, the command reuses and reveals it instead of creating duplicates.
14. The session prints an ephemeral cyan startup message in the form:

    Starting <model>...

15. The startup message is removed before the permanent launch confirmation is printed.
16. A permanent light-gray confirmation is printed in the form:

    * Launched <model>

17. The input prompt appears at the bottom after startup.
18. The helper text "/help for commands" is visible beneath or adjacent to the prompt without interfering with input.

Prompt behavior

19. The active input text is rendered in the normal terminal foreground color, treated as white for design purposes.
20. The prompt begins in chat mode using ">".
21. Typing "/" as the first input character switches the prompt into command mode.
22. In command mode, the prompt visually uses "/" rather than ">".
23. Pressing Escape while in command mode clears the command input and returns to chat mode.
24. Deleting the leading "/" returns the prompt to chat mode.
25. The cursor remains in the active input area at the bottom throughout normal interaction.
26. Submitted input is preserved in the transcript.
27. Submitted input is rendered in permanent light gray.
28. After submission, a fresh prompt appears at the bottom.

Command discovery

29. "/help" prints the available commands in permanent light gray.
30. "/help" also includes brief usage hints for:
    - Tab completion
    - Pressing Tab twice to list available commands
    - Escape to leave command mode
    - Up-arrow to recall prior input
31. Pressing Tab in command mode completes a command when there is a single unambiguous match.
32. Pressing Tab twice in command mode lists all available commands.
33. Command listing output is permanent light gray.
34. Unknown commands produce a permanent warning or error message and do not end the session.

Input history

35. Pressing Up Arrow recalls previously submitted chat inputs and commands.
36. Pressing Down Arrow navigates forward through recalled history where applicable.
37. Recalled input is editable before resubmission.
38. History remains available for the lifetime of the terminal session.

Ephemeral output

39. At most one ephemeral status line is visible at a time.
40. Ephemeral status text is rendered in cyan.
41. Ephemeral status may include a spinner or simple ASCII animation.
42. The ephemeral line may update in place while work is ongoing.
43. Before any permanent output is written, the current ephemeral line is completely removed.
44. Ephemeral text never remains in terminal scrollback as a completed transcript line.
45. No permanent line is written while an ephemeral line is still visible.
46. Starting a new ephemeral step replaces the prior ephemeral step rather than appending another temporary line.

Permanent output

47. Completed system output is rendered in light gray.
48. Final model output is rendered in light gray.
49. Completed system steps may be prefixed with "*" or another consistent completion marker.
50. Warnings are rendered in yellow and remain permanently visible.
51. Errors are rendered in red and remain permanently visible.
52. Warning and error meaning is not conveyed by color alone; each line must include "WARNING" or "ERROR".
53. Permanent output remains in terminal scrollback.
54. After permanent output is written, the active prompt remains at or returns to the bottom.

Chat submission

55. Entering non-command text submits it as a chat prompt.
56. The submitted chat text becomes permanent light-gray history.
57. While the model is responding, a cyan ephemeral status line is shown.
58. The status line includes visible activity, such as a spinner.
59. When the model response completes, the ephemeral line is removed.
60. The model response is printed inline in permanent light gray.
61. The session remains open for another prompt after the response.
62. A second prompt can be entered without relaunching the session.

Failure handling

63. If model startup fails, the ephemeral startup line is removed.
64. Startup failure is printed as a permanent red "ERROR" line.
65. If a chat request fails, the current ephemeral line is removed.
66. Chat failure is printed as a permanent red "ERROR" line.
67. If a chat response ends unexpectedly or is incomplete, a permanent warning or error is printed.
68. Failures do not leave the prompt area corrupted.
69. Recoverable failures do not automatically terminate the terminal session.

Exit and cancellation

70. Ctrl+C during active model work cancels the current operation.
71. Cancellation removes the ephemeral status line.
72. Cancellation prints a permanent light-gray or yellow cancellation message.
73. After cancellation, the prompt returns and the session remains usable.
74. "/exit" cleanly ends the session.
75. Exiting restores normal terminal behavior and does not leave a hanging child process.
76. Relaunching through "AI Dev: Launch Assistant" starts a fresh working session.

Compatibility and coexistence

77. Existing AI Dev commands remain available.
78. The terminal assistant is additive and does not replace the current UI in this ticket.
79. The implementation works in the project's supported VS Code version range beginning at 1.123.
80. The terminal remains usable in environments where ANSI color support is limited; prefixes and symbols still convey state.

TECHNICAL NOTES

Use a small, explicit rendering model:

type TerminalOutput =
  | { kind: "input"; text: string }
  | { kind: "ephemeral"; text: string }
  | { kind: "complete"; text: string }
  | { kind: "warning"; text: string }
  | { kind: "error"; text: string };

Recommended terminal state:

interface AssistantTerminalState {
  mode: "chat" | "command";
  input: string;
  history: string[];
  historyIndex: number;
  activeEphemeralText?: string;
  requestInFlight: boolean;
}

Core rendering invariant:

clear ephemeral
write permanent output
redraw prompt

Do not implement ephemeral behavior by recoloring prior transcript lines.
